  • The triple A testing method is a clear cut pattern that breaks down unit testing into 3 categories or "to-do's":

    1. Arrange: To arrange means to perform the setup of the test.

    2. Act: Take the necessary 'actions' or steps to perform the test.

    3. Assert: Verify the results of the testing.

  • Example Code:

    • def test_subtraction(value_a,value_b):

    """return value_a - value_b"""

    assert Subtraction.subtract(1,2) == -1

*The importance of testing your code is to make sure your code completes operations correctly and returns correct results. If your code is off and is not returning the results you are asserting, then you know there's a mistake in the code.*

External Testing: Travis CI

  • External testing is exactly what it sounds like: testing outside of python. In this article, we will use Travis CI as an external testing source. In the picture above is the best case scenario you want to see when you test your code on Travis.
  • Travis CI basically does what we had discussed earlier regarding linting files. Travis also checks all files for any errors and makes sure your code is written correctly.
  • Travis CI or Travis Continuous Integration builds and tests code changes in smaller increments to help give quick feedback about the change and if it was successful.
  • To deploy Travis, you need to create your code, link your code to GitHub or another program and Travis will automatically begin building your code and giving feedback.
  • The importance of external testing is like getting a second opinion from a doctor or in general: in case pylint missed any errors or erroneous code, Travis double checks and usually gives the same result as pylint, however, it can also become different outcomes. To be safe, external testing gives confidence that code is in great shape and changes work.
